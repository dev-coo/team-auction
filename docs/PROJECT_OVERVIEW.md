## 2. 프로젝트 배경

### 2.1 현재 상황
리그오브레전드를 비롯한 여러 팀 게임에서 팀장이 정해진 후 팀원을 뽑아야 하는 상황이 발생한다. 현재는 팀장끼리 수동으로 모여서 팀원을 뽑는 과정이 너무 복잡하고 명확한 규칙이 없어 진행이 어렵다.

**문제점:**
- 40명 정도의 참가자가 8팀을 구성하는 경매 절차에 2시간 이상 소요
- 체계적인 시스템 부재로 인한 혼란
- 실시간 정보 공유의 어려움

### 2.2 해결하고자 하는 문제
1. 팀장이 뽑을 팀원들의 순서를 한눈에 볼 수 있어야 함
2. 경매 대상인 팀원들의 능력을 볼 수 있어야 함
3. 각 팀장의 경매 포인트 현황을 실시간으로 확인할 수 있어야 함
4. 경매 진행 시 적절한 시간 분배가 되어야 함
5. 팀장, 옵저버가 동시에 화면을 보며 즐길 수 있어야 함
6. 경매 종료 후 결과 공표 및 대진표 생성 기능

---

## 3. 프로젝트 목적

### 3.1 주요 목적
- **실시간 경매 시스템**: 팀장이 보유 포인트로 팀원을 입찰하는 경매 시스템 제공
- **팀원 프로필 관리**: 주최자가 팀원의 포지션/능력을 사전 등록
- **실시간 커뮤니케이션**: 전체 채팅 운영

### 3.2 기대 효과
- 팀원의 능력을 한눈에 파악하여 전략적인 팀 조합 구성 가능
- 체계적인 시스템으로 경매 시간 단축 (2시간 → 목표: 1시간 이내)
- 볼거리가 풍부한 경매 경험 제공
- 경매 과정의 투명성 및 공정성 확보

## 4. 프로젝트 범위

### 4.1 포함 범위 (In Scope)
- 경매방 생성 및 설정 (주최자 기능)
- 팀장/팀원 정보 사전 등록 (주최자)
- 역할별 초대 링크 생성 (주최자, 팀장별, 옵저버)
- 실시간 경매 진행 (입찰, 타이머, 낙찰)
- 실시간 채팅 (전체 채팅)
- 경매 현황 대시보드 (포인트, 팀 구성 현황)
- 경매 결과 페이지

### 4.2 제외 범위 (Out of Scope)
- 회원가입/로그인 시스템 (링크 기반 참여로 대체)
- 결제/유료화 기능
- 모바일 앱 (웹 반응형으로 대체)
- 게임 API 연동 (추후 확장 고려)

## 5. 대상 사용자

### 5.1 주요 사용자

| 사용자 유형 | 설명 | 주요 니즈 |
|-------------|------|-----------|
| 주최자 | 경매를 생성하고 운영하는 사람 | 경매 타이틀, 팀장/팀원 사전 등록, 포인트 할당, 팀 인원 설정 등 전체 세팅 |
| 팀장 | 팀원을 뽑을 권한을 가진 사람 | 팀원들의 능력을 보고 전략적으로 입찰 |
| 팀원 | 경매 대상자, 팀장에게 뽑히는 사람 | 경매 관전 및 채팅 참여 (주최자가 사전 등록) |
| 옵저버 | 경매에 참여하지 않는 순수 관전자 | 경매 관전 및 채팅 참여 (공용 링크로 자유 입장) |

> **참고**: 주최자가 경매에 직접 참여하려면 팀원으로 등록해야 함 (팀장 불가)

### 5.2 사용자 시나리오

#### 주최자 시나리오
1. 롤 대회 개최를 위해 참가자를 사전에 모집 완료
2. 서비스에서 "경매 생성" 클릭
3. 경매 타이틀, 포인트, 팀 수 등 설정
4. 팀장 정보 등록 (이름, 포지션, 한줄소개)
5. 팀원 정보 등록 (이름, 포지션, 한줄소개)
6. 생성 완료 → 역할별 초대 링크 생성
   - 주최자 링크 1개
   - 팀장별 개별 링크 N개
   - 옵저버 공용 링크 1개 (팀원/관전자 공용)
7. 링크 배포 후 참가자 입장 대기
8. "경매 시작" 버튼으로 실시간 경매 시작
9. 경매 진행 컨트롤 (다음 버튼으로 순서 진행)

**경매 페이즈:**
```
WAITING (대기) → CAPTAIN_INTRO (팀장 소개) → SHUFFLE (팀원 셔플) → AUCTION (경매 진행) → FINISHED (종료)
```

**팀 이름 규칙:**
- 팀 이름은 `{팀장닉네임}팀` 형식으로 자동 생성 (예: "홍길동팀")

**유찰 처리:**
1. 1라운드에서 입찰 없이 유찰 시 → 2라운드 진행
2. 2라운드에서도 유찰 시 → 랜덤 배분 (잔여 포인트 많은 팀에 우선 배정)

#### 팀장 시나리오
1. 주최자로부터 본인 팀장 링크 수령 (팀별 고유 링크)
2. 링크 접속 → 주최자가 등록한 본인 정보 확인 → "본인 맞음" 확인
3. 경매방 입장
4. 주최자로부터 포인트 지급받음 (사전에 팀장들끼리 조율 완료)
5. 경매 대기
6. 팀원 경매 시작 시 필요한 팀원 파악
7. 최소단위 입찰 또는 직접입력 입찰로 경쟁
8. 경매 마무리

#### 옵저버 시나리오 (팀원/관전자 공용)
1. 주최자로부터 옵저버 공용 링크 수령
2. 링크 접속 → 닉네임 입력 → 입장
3. 경매 관전 및 전체 채팅 참여

> **참고**: 팀원(경매 대상)도 본인 경매를 관전하려면 옵저버로 입장

## 7. 경매 규칙

### 7.1 포인트 시스템
- **최대 포인트**: 1000 포인트 (주최자 설정에 따라 변동 가능)
- **포인트 지급**: 주최자가 각 팀장에게 사전 조율된 포인트 지급

### 7.2 입찰 단위 규칙
현재 입찰가에 따라 최소 입찰 단위가 동적으로 변경됨:

| 현재 입찰가 | 최소 입찰 단위 |
|-------------|----------------|
| 0 ~ 99 | 5 포인트 |
| 100 ~ 199 | 10 포인트 |
| 200 ~ 299 | 15 포인트 |
| 300 ~ 399 | 20 포인트 |
| ... | +5씩 증가 |

### 7.3 경매 타이머 규칙
- **초기 시간**: 팀원 1명당 15초
- **입찰 시 시간 추가**: 입찰이 발생하면 남은 시간 + 2초
- **낙찰 조건**: 타이머가 0초가 되면 마지막 입찰자에게 낙찰

**타이머 예시:**
```
15초 시작
 ↓ (5초 경과)
10초 남음 → A팀장 5포인트 입찰
 ↓ (+2초 추가)
12초로 리셋
 ↓ (3초 경과)
 9초 남음 → B팀장 10포인트 입찰
 ↓ (+2초 추가)
11초로 리셋
 ↓ (타이머 종료까지 입찰 없음)
낙찰: B팀장
```

## 8. 제약 사항

### 8.1 기술적 제약
- 실시간 통신 필수 (WebSocket 또는 유사 기술)
- 동시 접속자 처리 (최소 50명 이상)
- 모바일 브라우저 호환성 확보 필요

### 8.2 비즈니스 제약
- 비상업적 목적 (개인/커뮤니티 프로젝트)
- 초기 버전은 리그오브레전드 중심으로 개발 후 확장 고려

## 9. 성공 기준

- [ ] 40명 규모의 경매를 1시간 이내에 완료 가능
- [ ] 모든 참가자가 동시에 경매 현황을 실시간으로 확인 가능
- [ ] 팀장 간 입찰 경쟁이 원활하게 진행됨
- [ ] 낙찰 처리가 정상 작동하고 팀 구성 현황에 즉시 반영됨
- [ ] 경매 종료 후 결과 확인 및 공유 가능
